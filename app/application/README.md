# Application Layer 🧠

This folder contains the business logic and application services that implement the core functionality of the ESP32 voice assistant.

## 📋 Application Components

### GPT Service (`gpt_service.h/.cpp`)
**Purpose**: AI-powered conversation service using OpenAI GPT models for natural language processing and response generation.

**Key Features**:
- **Multi-Model Support**: Configurable GPT models with cost optimization
- **Conversation Caching**: Efficient memory management for chat history
- **Error Handling**: Robust API error handling and retry logic
- **Affordable Models**: Automatic selection of cost-effective models

### Voice Display Coordinator (`voice_display_coordinator.h/.cpp`)
**Purpose**: Coordinates voice recognition events with display state management for seamless user interaction.

**Key Features**:
- **Event Coordination**: Synchronizes voice commands with visual feedback
- **State Management**: Maintains display state across voice interactions
- **Real-time Updates**: Immediate visual responses to voice commands</content>
